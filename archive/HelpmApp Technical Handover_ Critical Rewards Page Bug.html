HelpmApp Technical Handover: Critical Rewards Page Bug ​

To: HelpmApp Coding Agent From: Manus AI (Project Manager) Date: December 28, 2025 Priority: CRITICAL - P0 (Directly impacts core value proposition and funding narrative)

1. Issue Summary ​

The Rewards page, accessible via the top navigation link, is currently non-functional on the production environment. Instead of displaying the rewards catalogue or a login prompt, it throws a client-side exception.

This bug must be fixed immediately as the rewards system is a core component of the HelpmApp's gamification strategy and is a key selling point for corporate sponsors and grant applications.

2. Reproduction Steps ​

  1. Navigate to the production URL: https://www.helpmapp.org
  2. Click the "Rewards" link in the top navigation bar.
  3. Expected Result: The user should be taken to the /rewards route, which should display either the rewards catalogue (if logged in) or a login/sign-up prompt (if logged out).
  4. Actual Result: The page fails to load and displays the message:

    "Application error: a client-side exception has occurred while loading www.helpmapp.org (see the browser console for more information)."

3. Technical Context & Hypothesis ​

  • Route: /rewards
  • Likely Cause: Given the Vercel error message, the issue is almost certainly a client-side rendering (CSR) error within the React component for the Rewards page. This could be due to:
    • An unhandled null or undefined state when fetching rewards data (e.g., the component assumes data is present but the API call fails or returns an empty object).
    • A missing environment variable or configuration setting required by the Rewards component in the production environment.
    • A dependency mismatch or a side effect in useEffect that is not correctly guarded.
  • Impact: This failure directly contradicts the "de-risked build" narrative being presented to funders.

4. Proposed Solution (Action for Coding Agent) ​

  1. Check Console Logs: Review the browser console logs (or Vercel logs) for the specific exception stack trace to pinpoint the failing line of code in the Rewards component.
  2. Implement Robust Error Handling: Wrap the data fetching and rendering logic in the Rewards component with try...catch blocks and conditional rendering checks (e.g., if (!rewardsData) return <LoadingState />).
  3. Verify Environment Variables: Ensure all necessary environment variables (e.g., API keys, database connection strings for the rewards ledger) are correctly configured in the Vercel production environment.
  4. Temporary Fix (If necessary): If the full fix requires extensive work, deploy a temporary component that simply displays a message like: "Our rewards catalogue is currently being updated. Please check back soon!" This is a higher priority than the full fix, as it removes the critical error from the user experience.

5. Next Steps ​

  • Coding Agent: Acknowledge receipt and provide an estimated time to fix (ETA).
  • PM (Manus AI): Will monitor the Vercel deployment status for the fix.

Please prioritize this fix. A fully functional MVP is essential for the upcoming grant applications.
