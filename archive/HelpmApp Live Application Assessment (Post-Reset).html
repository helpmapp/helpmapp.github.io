HelpmApp Live Application Assessment (Post-Reset) ​

Author: Manus AI Date: December 28, 2025 Source Documents: HelpmApp Documentation Suite (Version 2.0, Nov 2025) Live Application URL: https://www.helpmapp.org

1. Executive Summary ​

The live HelpmApp application at https://www.helpmapp.org demonstrates a high degree of alignment with the strategic vision outlined in the Version 2.0 documentation suite. The core service discovery functionality is robust, presenting a map-based interface with filtering by category and preset locations. This validates the foundational technical and product decisions made during the "clean-slate rebuild."

However, the assessment reveals a critical inconsistency: the Rewards page is currently non-functional, displaying a client-side error. Since the gamified rewards system is a core pillar of the HelpmApp's differentiation and funding narrative, this bug must be addressed immediately to maintain the "de-risked" status presented to funders.

2. Alignment with Documentation and Roadmap ​

The live application successfully implements the core features described in the Operational Plan and Technical RFC for the initial Foundation (Months 1-6) phase.

3. Strategic Recommendations ​

Based on the live application's status, the following strategic adjustments are recommended to maintain momentum and strengthen the funding narrative:

Recommendation 1: Immediate Fix for the Rewards Page Bug ​

The non-functional Rewards page undermines the core value proposition of "Gamified Engagement" [2].

  • Action: Prioritize a hotfix for the /rewards route. Even a temporary placeholder page explaining the rewards system is preferable to a client-side error, especially if the live URL is being shared with potential funders or partners.
  • Rationale: The £70,000 funding ask is predicated on the rewards pool and user acquisition [5]. A broken rewards page directly contradicts the "de-risked build" messaging.

Recommendation 2: Leverage the "270 Services" Metric ​

The live app displays 270 Services, significantly higher than the 64+ services mentioned in the legacy traction section of the Pitch Deck [4].

  • Action: Update all external-facing documentation, including the Pitch Deck and Executive Summary, to reflect the 270 Services metric. This demonstrates stronger current traction and a more robust initial data set.
  • Action: Ensure the Impact Measurement Framework is fully instrumented to track the quality and completeness of these 270 services [6].

Recommendation 3: Finalize Technical Decisions (DB and Map Stack) ​

The live application is clearly using a map component, which makes the Map Provider decision (Mapbox vs. Google Maps) urgent, as licensing costs can be significant.

  • Action: Finalize the database decision (recommending Supabase with PostGIS for geospatial needs) and the map provider to lock in the technical foundation for the next phases of development (e.g., check-in verification, analytics dashboards).

By addressing the critical Rewards bug and updating the documentation to reflect the strong current traction, HelpmApp can confidently proceed with its aggressive funding and operational roadmap.
